ADI_FEE_RESULT_TYPE adi_FEE_GetBlockNumber ( ADI_FEE_DEV_HANDLE const  hDevice,
const uint32_t  Address,
uint32_t *  pnBlock 
)

Gets a block number as a bitmask.

get flash block number

Parameters
[in]hDeviceHandle of flash device
[in]nAddressMemory address to find out which block it belongs to
[out]*pnBlockBlock mask output argument
Returns
Status
  • ADI_FEE_SUCCESS if successfully initialized
  • ADI_FEE_ERR_INVALID_ADDRESS if address out of range

Converts an absolute flash memory address to a number identifying a specific block within the specified controller's flash region. Since there are always 32 blocks per region the block number is represented as a bitmask with one bit set. This allows multiple block numbers to be ORed together to express a set of blocks.

ADI_FEE_RESULT_TYPE adi_FEE_Init ( ADI_FEE_DEV_ID_TYPE const  devID,
bool_t  bRetryAborts,
ADI_FEE_DEV_HANDLE *const  phDevice 
)

Initialize and allocate an FEE device.

initialize flash driver

Parameters
[in]devIDZero-based device index designating which Flash device to initialize.
[in]bool_ttrue if driver should retry on flash aborts, false to return errors
[out]phDeviceThe caller's device handle pointer for storing the initialized device instance data pointer.
Returns
Status
  • ADI_FEE_ERR_ALREADY_INITIALIZED [D] Specified device has already been initialized.
  • ADI_FEE_ERR_BAD_DEV_ID [D] Invalid device index.
  • #ADI_FEE_ERR_SEMAPHORE_FAILED Failed to create semaphore.
  • #ADI_FEE_ERR_DMA_FAILURE Failed to initialize DMA.
  • ADI_FEE_SUCCESS Call completed successfully.

No other FEE APIs may be called until the device init function is called.

Initialize an FEE device using internal default configuration settings and allocate the device for use. The caller's device handle is written with the device instance data pointer. The returned device handle is required to be passed to all subsequent calls to convey which device instance to operate on.

Note
The contents of phDevice will be set to NULL upon failure.

ADI_FEE_RESULT_TYPE adi_FEE_MassErase ( ADI_FEE_DEV_HANDLE const  hDevice)

Mass Erase.

erase entire flash

This function will erase the entire contents of the Flash. Applications that use this API should be configured to execute from SRAM, otherwise this routine will effectively erase itself.

Parameters

Returns
Status

  • #ADI_FEE_ABORTED - operation was aborted
